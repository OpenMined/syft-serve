Why We Built This

Managing multiple FastAPI servers shouldn't require DevOps expertise. SyftServe makes it trivial.

üöÄ Launch in Seconds

From Python function to running server in one line. No config files, no boilerplate, no complexity.

üì¶ Isolated Environments

Each server gets its own virtual environment. Install pandas in one, tensorflow in another - no conflicts!

üìù Named Servers

The Whole API

Yes, this is literally everything

Creating Servers

# Create a server
server = ss.create(
    name="my_api",
    endpoints={"/hello": hello_func},
    dependencies=["pandas>=2.0.0"]  # Optional
)

Accessing Servers

# View all servers
ss.servers                    # Interactive table

# Get specific server
server = ss.servers["my_api"] # By name
server = ss.servers[0]        # By index

Server Properties

# Basic info
server.name       # "my_api"
server.url        # "http://localhost:8000"
server.port       # 8000
server.status     # "running"
server.endpoints  # ["/hello"]
server.uptime     # "2m 15s"

Viewing Logs

# Access logs easily
server.stdout.tail(20)   # Last 20 lines
server.stdout.head(10)   # First 10 lines
server.stdout.lines()    # All lines as list
server.stderr.tail(10)   # Error logs

Environment Info

# Check installed packages
server.env               # Pretty tree view
server.env.list()        # List all packages

Managing Servers

# Replace existing server
server = ss.create(
    name="my_api",
    endpoints={"/v2": new_func},
    force=True  # Replaces existing
)

# Terminate specific server
server.terminate()

# Terminate all servers
ss.terminate_all()

Multiple Endpoints

# Server with multiple endpoints
api = ss.create(
    name="full_api",
    endpoints={
        "/users": get_users,
        "/posts": get_posts,
        "/analytics": get_analytics
    }
)

Persistence

# Servers survive notebook restarts!

import syft_serve as ss

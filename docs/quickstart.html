1-Minute Quick Start

From zero to running servers in 60 seconds.

1 Install SyftServe

pip install syft-serve

2 Launch Your First Server

Create a simple API endpoint and launch it:

import syft_serve as ss

# Define an endpoint
def hello():
    return {"message": "Hello from SyftServe!"}

# Create the server
server = ss.create(
    name="hello_api",
    endpoints={"/hello": hello}
)

print(f"Server running at {server.url}")
# Output: Server running at http://localhost:8000

3 Test Your Server

Your server is already running! Test it with requests:

import requests

# Make a request to your endpoint
response = requests.get(f"{server.url}/hello")
print(response.json())
# Output: {'message': 'Hello from SyftServe!'}

4 View & Access Servers

See all your running servers in a nice table:

# View all servers
ss.servers

# Output:
# Name         Port  Status     Endpoints  Uptime    PID
# ----------  ------  ---------  ---------  ------  -----
# hello_api    8000  âœ… Running  /hello     45s     1234

# Access a specific server
my_server = ss.servers["hello_api"]
print(f"Uptime: {my_server.uptime}")
print(f"Recent logs: {my_server.stdout.tail(5)}")

5 Real-World Example

Here's how a data science team might use SyftServe:

# Data processing API
def process_data():
    import pandas as pd
    # Your data processing logic here
    return {"status": "processed", "rows": 1000}

data_api = ss.create(
    name="data_processor",
    endpoints={"/process": process_data},
    dependencies=["pandas", "numpy"]  # Auto-installed!
)

# ML prediction API
def predict():
    # Your ML model logic here
    return {"prediction": 0.95, "confidence": 0.87}

ml_api = ss.create(
    name="ml_predictor",
    endpoints={"/predict": predict},
    dependencies=["tensorflow", "scikit-learn"]
)

# View all your services
ss.servers  # Shows both servers running on different ports
